# Классификатор отзывов (NLP)

Проект по обучению и развертыванию модели **intfloat/multilingual-e5-small** для **поиска негативных отзывов** с использованием **NVIDIA Triton Inference Server**.

## Цель проекта

**Задача:** Автоматическая классификация текстовых отзывов для **выявления негативных обращений** (бинарная классификация: негативный / позитивный отзыв).

---

## Ключевые особенности

- Fine-tuning модели E5 для бинарной классификации
- Предобработка и анализ датасета [`geo-reviews-dataset-2023.csv`](https://www.kaggle.com/datasets/kyakovlev/yandex-geo-reviews-dataset-2023)
- Подробные материалы по работе токенизатора
- Экспорт модели в ONNX формат
- Развертывание Triton Server через Docker Compose
- gRPC клиент для инференса

---

## Структура проекта

### `triton/models/` - Папка, которая монтируется как volume в Triton Server. 
- **`bert_classifier/`** — ONNX модель (`model.onnx`) + конфиг (`config.pbtxt`)
- **`text_tokenizer/`** — токенизатор с Python бэкендом (`model.py`)
- **`text_classifier_ensemble/`** — ансамбль для единого пайплайна

### Корневые файлы
- **`train_notebook.ipynb`** — полный цикл: предобработка датасета → токенизация → обучение E5 → конвертация в ONNX
- **`test_triton.ipynb`** — gRPC клиент для тестирования
- **`docker-compose.yaml`** — запуск Triton Server
- **`best_model.pth`** — веса обученной модели

---

## Как запустить проект

**Запуск тритона с уже обученной моделью:**
```bash
docker-compose up --build   
```

**Обучение:** Откройте train_notebook.ipynb — там предобработка данных, объяснение токенизатора, обучение и экспорт в ONNX.

**Тестирование поднятого тритона:** Запустите test_triton.ipynb для gRPC запросов к серверу.
